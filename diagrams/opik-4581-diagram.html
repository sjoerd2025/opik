ASSERTIONRESULTMAPPER.ENRICHWITHASSERTONS()
1
Partition feedback_scores by category_name = "suite_assertion"
2
Map assertion scores → AssertionResult DTOs (value, passed, reason)
3
Compute per-run status: all passed → "passed", else "failed"
4
Remove assertion scores from feedbackScores list
ASSERTIONRESULTMAPPER.COMPUTERUNSUMMARIES()
1
Groups items by experimentId
2
Counts passedRuns per experiment
3
Reads passThreshold from executionPolicy
4
Returns Map<String, ExperimentRunSummary> set on DatasetItem
RESPONSE STRUCTURE (OPTION B)
DatasetItem {
  experiment_items: [
    {
      experiment_id: "exp-A",
      status: "passed",  // per-run: all assertions passed?
      assertion_results: [
        { value: "Should link to docs", passed: true, reason: "Links found" }
      ],
      feedback_scores: [ ... ]  // assertion scores removed
    }
  ],
  run_summaries_by_experiment: {
    "exp-id-A": {
      passed_runs: 3,
      total_runs: 5,
      status: "failed"  // passedRuns >= passThreshold?
    }
  }
}
!
execution_policy is @JsonIgnore on ExperimentItem — used internally, not in response
